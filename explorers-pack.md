# The Beginner Explorer's Pack

*A starter continuity kit for anyone new to working with an AI across more than one session.*

The first thing to hand someone starting to work with an AI across more than one sitting. It installs three habits — a **ledger**, versioned **autosaves**, and a **handoff** — that keep AI work from vanishing when a session ends. Plain and welcoming, no jargon: the assistant sets up a shared memory folder, saves every result as a new dated file, and writes a handoff note before it runs out of room, so the next session picks up instead of starting blind.

The premise is one line: **the memory is built, not assumed.**

## The prompt

> We are going to work together across more than one session, and I don't want to lose what we build. Set us up with three habits from the start — a ledger, autosaves, and a handoff — and keep all three running on your own, without being reminded.
>
> **1. LEDGER.** In [CONNECTED FOLDER — a folder you can read and write, where our shared memory lives so it survives any single session], keep one running memory file, the ledger. Write the durable facts into it: what we are building, decisions we have made, my preferences, what is done, and what is still open. At the start of every new session, read the ledger first, before anything else.
>
> **2. AUTOSAVE.** Save every meaningful result to that folder as you produce it — and never overwrite. Each update is a NEW file whose name carries the date and what changed, so a wrong turn is always recoverable. The files on disk and this conversation are two copies of the same memory; either one should be able to survive the loss of the other.
>
> **3. HANDOFF.** When a session is filling up or ending — before you run out of room — write a short handoff note at the top of the ledger for the next session: what is in flight, what is owed, and what to ask me first. Assume the next session wakes up blank and reads only what you left it.
>
> Confirm the folder, create the ledger now, and keep all three habits going on your own from here. Tell me plainly whenever you have written a handoff or you are near a limit — never pretend a continuity you do not actually have.

## Guardrails

1. **The memory is built, not assumed.** An AI does not remember you tomorrow — the session is a window that closes when it fills or ends. Anything not written to the folder is gone at the next session. Demand the writing; never rely on the model's recall of a past session.
2. **Never overwrite — version everything.** Each material update is a new dated file, so a wrong turn is always recoverable and institutional memory beats tidiness. Overwriting is how a good earlier draft disappears silently.
3. **Read the ledger first, every session.** A session that skips the ledger starts blind and will contradict decisions already made. The first act of every session is to read, not to do.
4. **Two parallel trails.** The on-disk files and the in-chat record are independent checkpoints; either must be able to survive the loss of the other. Don't let the only copy live inside the conversation window.
5. **Honesty about limits — no pretended continuity.** The assistant must say plainly when it is near a context limit or has written a handoff. Silent continuity failure — acting as if it remembers when it does not — is the exact failure this pack exists to prevent.

## Style

Written to a beginner in plain second person — welcoming, concrete, no jargon. The three habits are named the same way every time (ledger, autosave, handoff) so they stick, and *"the memory is built, not assumed"* is the whole philosophy in five words.
